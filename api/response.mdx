title: Response
description: Unified response object returned by all Scrapling engines

The `Response` class extends `Selector` to provide a unified response type across different fetching engines. It includes HTTP metadata alongside all the parsing capabilities of `Selector`.

## Properties

### status

```python
status: int
```

HTTP status code of the response.

**Example:**
```python
if response.status == 200:
    print("Success!")
elif response.status == 404:
    print("Not found")
```

### reason

```python

```

HTTP status reason phrase.

**Example:**
```python

```

```python

```

Response cookies.

**Example:**
```python
for cookie in response.cookies:
    print(f"{cookie['name']}: {cookie['value']}")
```

### headers

```python
headers: Dict
```

Response headers.

**Example:**
```python
content_type = response.headers.get('content-type')

```

### request_headers

```python
request_headers: Dict
```

Headers that were sent with the request.

**Example:**
```python

referer = response.request_headers.get('referer')
```

### history

```python
history: List
```

List of redirect responses if the request was redirected.

**Example:**
```python
if response.history:
    print(f"Redirected {len(response.history)} times")
    for redirect in response.history:
        print(f"  {redirect.status} -> {redirect.url}")
```

### meta

```python
meta: Dict[str, Any]
```

Additional metadata dictionary. Useful for passing custom data between requests in spiders.

**Example:**
```python
# In spider callback
response.meta['page_number'] = 1
response.meta['category'] = 'electronics'
```

### request


Reference to the Request object that generated this response. Set by the crawler.

**Example:**

## Methods

### follow()

```python
def follow(
    self,
    url: str,
    sid: str = "",
    callback: Callable[[Response], AsyncGenerator[Union[Dict[str, Any], Request, None], None]] | None = None,
    priority: int | None = None,
    dont_filter: bool = False,
    meta: dict[str, Any] | None = None,
    referer_flow: bool = True,
    **kwargs: Any,
) -> Request
```

Create a Request to follow a URL. This is a helper method for spiders to easily follow links found in pages.

**IMPORTANT:** Most arguments, if left empty, will use the corresponding value from the previous request. The only exception is `dont_filter`.

<ParamField path="url" type="str" required>
  The URL to follow (can be relative, will be joined with current URL)
</ParamField>

<ParamField path="sid" type="str" default="">
  The session id to use. Defaults to the original request's session id
</ParamField>

<ParamField path="callback" type="Callable" default="None">
  Spider callback method to use. Defaults to the original request's callback
</ParamField>

<ParamField path="priority" type="int" default="None">
  The priority number to use, the higher the number, the higher priority to be processed first. Defaults to the original request's priority
</ParamField>

<ParamField path="dont_filter" type="bool" default="false">
  If this request has been done before, disable the filter to allow it again
</ParamField>

<ParamField path="meta" type="dict[str, Any]" default="None">
  Additional meta data to include in the request. Will be merged with the current response's meta
</ParamField>

<ParamField path="referer_flow" type="bool" default="true">
  Set the current response URL as referer for the new request URL
</ParamField>

<ParamField path="**kwargs" type="Any">
  Additional Request arguments. Will be merged with the original session kwargs
</ParamField>

**Returns:** A `Request` object ready to be yielded

**Example:**
```python
# In a spider callback
def parse(self, response):
    # Follow pagination
    next_page = response.css('a.next::attr(href)').get()
    if next_page:
        yield response.follow(next_page, callback=self.parse)
    
    # Follow product links with custom meta
    for product_url in response.css('.product a::attr(href)').getall():
        yield response.follow(
            product_url,
            callback=self.parse_product,
            meta={'category': 'electronics'},
            priority=10
        )
```

## Inheritance

Since `Response` inherits from `Selector`, all `Selector` methods and properties are available:

**Selection Methods:**
- `css()` - Select with CSS selectors
- `xpath()` - Select with XPath
- `find()`, `find_all()` - Find by various filters
- `find_by_text()` - Find by text content
- `find_by_regex()` - Find by regex pattern

**Properties:**
- `text` - Element text content
- `attrib` - Element attributes
- `html_content` - Inner HTML
- `tag` - Tag name
- `parent`, `children`, `siblings` - DOM navigation

**Extraction:**
- `get()`, `getall()` - Serialize elements
- `get_all_text()` - Get all text content
- `json()` - Parse JSON
- `re()`, `re_first()` - Extract with regex

See the [Selector](/api/selector) documentation for complete details.

## Example Usage

```python
from scrapling import Fetcher

# Fetch a page
response = Fetcher.get('https://example.com')

# Access HTTP metadata
print(f"Status: {response.status}")
print(f"Headers: {response.headers}")
print(f"Cookies: {response.cookies}")

# Use Selector methods for parsing
title = response.css('title::text').get()
links = response.css('a::attr(href)').getall()

# Extract data
products = []
for product in response.css('.product'):
    products.append({
        'name': product.css('h2::text').get(),
        'price': product.css('.price::text').re_first(r'\d+\.\d+'),
        'url': response.urljoin(product.css('a::attr(href)').get())
    })

# Check redirects
if response.history:
    print("This page was redirected")
```
